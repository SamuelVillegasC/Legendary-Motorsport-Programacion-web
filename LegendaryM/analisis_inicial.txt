1. Requerimientos Funcionales

A. Gestión de Catálogo y Navegación
Filtrado Dinámico: El sistema debe permitir organizar los vehículos por categorías (Súper, Deportivos, Clásicos, Motos) y por criterios de precio (Ascendente/Descendente).

Fichas de Producto Detalladas: Cada vehículo debe mostrar una galería de imágenes, descripción técnica de "lore" y una tabla de estadísticas de rendimiento (Velocidad, Aceleración, Frenado y Tracción).

Sistema de Stock Real: El backend debe validar si el vehículo está disponible o si aparece como "Agotado" (Out of Stock) basado en eventos de temporada o inventario global.

B. Módulo de Personalización Visual (Pre-compra)
Selector de Color: Antes de añadir al carrito, el usuario debe poder seleccionar el color de la carrocería entre una paleta predefinida. Esto debe actualizar el renderizado de la imagen en tiempo real.

C. Procesamiento de Transacciones
Validación de Fondos: Integración con la API del banco (Maze Bank) para verificar que el usuario tiene el saldo suficiente antes de procesar la orden.

Confirmación de Entrega: Una vez confirmada la compra, el sistema debe disparar un evento de logística para asignar el vehículo al garaje seleccionado por el usuario.

2. Requerimientos No Funcionales

A. Rendimiento y Escalabilidad
Tiempo de Carga: Siendo una web de lujo, el despliegue de imágenes de alta resolución no debe penalizar el tiempo de carga inicial. Se requiere el uso de Lazy Loading y formatos de imagen optimizados (como WebP).

Alta Disponibilidad: La plataforma debe soportar picos de tráfico masivos cuando se anuncian nuevos lanzamientos (DLCs) sin degradar el servicio.

B. Experiencia de Usuario (UX/UI)
Estética "Premium-Dirty": La interfaz debe ser minimalista pero agresiva, utilizando una paleta de colores oscuros con acentos dorados o blancos para resaltar el estatus.

Responsividad: Aunque en el juego se ve en un navegador fijo, el diseño debe ser Mobile-First para que el usuario pueda comprar desde su iFruit en cualquier lugar del mapa.

C. Seguridad
Prevención de Inyecciones: Protección estricta contra ataques que intenten alterar los precios de los vehículos desde la consola del navegador.

Integridad de Datos: Asegurar que la transacción sea atómica: o se descuenta el dinero y se entrega el coche, o no ocurre nada (evitar que el usuario pierda dinero por un glitch).
